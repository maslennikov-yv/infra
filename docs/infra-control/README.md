# Спецификация infra-control

**infra-lab** — точка входа для полного жизненного цикла инфраструктуры и приложений в репозитории: среда, сервисы, приложения и их конфигурация. Интерактивная навигация без зубрёжки длинных вызовов; выбранные действия выполняют те же именованные сценарии, что и в корне репозитория, чтобы вызовы оставались согласованными и воспроизводимыми.

## Запуск

```bash
cd /path/to/infra && npm ci && node scripts/infra-lab.mjs
```

Стек промптов — [@clack/prompts](https://github.com/bombshell-dev/clack) (`package.json`). Реализация: каталог [`scripts/infra-control/`](../../scripts/infra-control/) и точка входа [`scripts/infra-lab.mjs`](../../scripts/infra-lab.mjs). В каждом меню есть пункт **«Справка»** (перед «Назад» или «Выход»): по нажатию показывается контекст раздела и расшифровка имён целей. Корневое меню группирует действия по **задаче** (бутстрап, конфигурирование, управление) и **объекту** (среда, сервис, приложение); подробнее — [menu-tree.md](menu-tree.md).

## Артефакты спецификации

| Файл | Назначение |
|------|-------------|
| [menu-tree.md](menu-tree.md) | Иерархия TUI: задача × объект и подразделы |
| [targets.json](targets.json) | Машиночитаемое соответствие `menuPath ↔ имя цели` (+ `dangerous`, `sigint`). Цели мета-сверки **`infra-control-parity-check`** в manifest не включаются. |
| [acceptance.md](acceptance.md) | Definition of Done и ручные сценарии приёмки |
| [_generate-targets-manifest.py](_generate-targets-manifest.py) | Регенерация `targets.json` при больших изменениях `.PHONY` (ручной запуск) |

Проверка паритета объявленных целей и manifest:

```bash
node docs/infra-control/parity-check.mjs
```

(В корне репозитория есть и отдельная цель для той же проверки — см. описание целей в README репозитория.)
