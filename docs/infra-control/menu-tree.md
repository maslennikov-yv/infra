# Дерево меню infra-control

Интерактивное TUI: `npm install && node scripts/infra-lab.mjs` (зависимость `@clack/prompts`). Паритет с **именами целей и переменными** в репозитории — не по тексту подсказок в исходниках сценариев.

Навигация **двухуровневая**: сначала **задача** (бутстрап, конфигурирование, управление), затем **объект** (среда, сервис, приложение; при конфигурировании первым идёт **сессия**). Исключения: **Справка** и **Выход** — с первого уровня. Для **конфигурирования** объект **«Сервис»** отсутствует (правки values — в каталогах чартов и git).

**Подписи пунктов** в TUI — краткое **описание действия** на русском; выбранное **имя цели** показывается в логе перед запуском.

Принципы:

- В подзаголовках указывается активный **`ENV`** (сессия). Для действий через SSH — **`SSH_HOST`** / `environments/$(ENV).mk`.
- Лист дерева вызывает **одну** именованную цель (иногда после сбора `VAR=value` через промпты).
- Опасные цели — два шага подтверждения или ввод ключевого слова перед запуском.
- Отмена (Ctrl+C / отмена Clack) ведёт к корректному выходу.

Машиночитаемые пути `menuPath` см. [targets.json](targets.json) (префикс: задача, объект, затем прежняя иерархия раздела). Полный перечень подписей — в исходнике [`scripts/infra-control/run.mjs`](../../scripts/infra-control/run.mjs). Тексты подсказок для новичка вынесены в пункт **«Справка»** в интерфейсе (не дублируются в этом документе).

## Уровень 1: задача

1. **Бутстрап**
2. **Конфигурирование**
3. **Управление**
4. **Справка**
5. **Выход**

---

## Бутстрап

### Объект «Среда»

- Создать скелет окружения (файлы, values для сервисов).
- Доступ к кластеру: скачать kubeconfig, SSH, установка/проверка MicroK8s.

### Объект «Сервис»

- Инициализация подключения к Kafka (bootstrap).
- Развернуть весь стек Helm либо выбрать сервисы.

### Объект «Приложение»

- Клонировать репозиторий приложения; каталог из шаблона `apps/conf`; **конфигуратор** (секреты, registry) — см. [`scripts/configure-infra.mjs`](../../scripts/configure-infra.mjs).

---

## Конфигурирование

### Объект «Сессия»

- Смена активного ENV для сеанса infra-lab (профиль `environments/<ENV>.yaml`, kubeconfig; напоминание про `environments/<ENV>.mk`).

### Объект «Среда»

- Показать или изменить проброс TCP / hostPort (microk8s ingress).

### Объект «Приложение»

- Показать итоговую конфигурацию (merge в stdout); вывести `helm --set` для hostPath приложения при `ENV=local` (`apps-local-src-helm-sets`); конфигуратор.

---

## Управление

### Объект «Среда»

- Кластер: обзор, нагрузка узлов, kubeconfig, SSH, MicroK8s, удаление кластерного ПО (опасно).
- Архив секретов и манифестов окружения.
- Образы: сохранение tar, push в registry, push на удалённом хосте.
- Helm для **всего набора**: развернуть, сравнить, уничтожить.

### Объект «Сервис»

- Helm: сравнение всего набора и поштучные действия по компоненту.
- Диагностика (логи, shell, данные Postgres и т.д.).
- Проверки чартов и обновлений (динамический список целей).
- Kafka: сброс данных, операции с топиками (без создания по APP на ветке «сервис»).

### Объект «Приложение»

- Применить учётки и конфиги приложений в кластер.
- Логины приложений в БД и брокерах (по движкам).
- Топики Kafka с привязкой к приложению.

---

## Длинные формы и «ещё параметры»

Как ранее: сложные цели — короткий путь обязательных полей и меню **Доп. переменные окружения** (`KEY=value`). Листья строят `env` для **`runMake`** (см. [`scripts/lib/make.mjs`](../../scripts/lib/make.mjs)).
